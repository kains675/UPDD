### 7.0 UPDD's Differentiating Contribution: ncAA Expansion via Stage 4 Evaluation

UPDD's primary differentiating contribution is its **Stage 4 evaluation framework for ncAA-modified peptide complexes**, integrated with established generation tools (RFdiffusion + ProteinMPNN + AF2) within a 5-stage iterative pipeline (per §4.1.2).

**What this means concretely**:
- Generation (Stages 2-3) leverages field-standard tools without modification
- UPDD's novelty concentrates in Stage 4: ncAA parameterization + MD + QM/MM + MM-PBSA with σ_btwn vs σ_w decomposition diagnostics
- The evaluation framework is built around honest handling of ncAA charge / parameterization gaps, not around masking them
- Iterative cycle architecture (Cycle 0 / Cycle 1+, per §4.1.3) enables wet-lab feedback ingestion without disturbing upstream sampling state

**Why this framing matters for Paper 1**:
- Avoids over-claiming on generation novelty (which UPDD does not provide)
- Concentrates the claim on the verifiable, reproducible Stage 4 contribution
- Aligns paper narrative with Capability Level 1 (decision support) — honest about scope
- Positions UPDD as a complement to generation tools, not a replacement

**Cross-references**:
- Stage definitions: §4.1.2
- Iterative cycle architecture: §4.1.3
- Capability Levels: §4.1.1

### 7.1 Primary Scientific Contribution ⭐

**The Stage 4 evaluation framework for ncAA peptide complexes** (per §4.1.2 + §7.0) is UPDD's primary contribution. Within this framework, the **σ_btwn vs σ_w decomposition** (§3.3) is the most generalizable methodological finding of Phase α and serves as the framework's signature diagnostic.

**The σ_btwn vs σ_w decomposition framework** (§3.3) is Phase α's most generalizable scientific finding.

**Core claim**:
> "Charge correctness in MM-PBSA pipelines is not merely numerical accuracy
> but a prerequisite for genuine ensemble exploration. Wrong charges can
> produce narrow-but-divergent artifact ensembles that present as 'low noise'
> under within-replicate analysis but reveal divergence under between-replicate
> scrutiny. The σ_btwn vs σ_w decomposition is necessary to distinguish
> artifact-narrow from genuine-broad sampling."

**Why this is independently publishable**:
- Generalizable beyond UPDD: applies to any MM-PBSA pipeline with ncAA charge handling
- Methodologically rigorous: two orthogonal axes constraining mechanism interpretation
- Directly testable: other groups can apply to their data without UPDD code
- Field-novel: charge errors are common but rarely tracked at ensemble decomposition level
- Dual-system evidence: Cp4 + 1EBP_MTR13 (this work)

**Recommended paper title approach**: Lead with the Stage 4 ncAA evaluation framework (§7.0) as the primary contribution, with σ_btwn vs σ_w decomposition as its signature diagnostic — not with UPDD-specific Cp4/1EBP results in isolation.

### 7.2 Methodology Contribution: Multi-Layer Verification Protocol

The two halt-and-recovery events (§4.6) provide **empirical validation** of multi-layer verification design — a project management contribution to computational chemistry methodology.

**Novel angle**:
- Most computational chemistry tools rely on test-pass-only validation
- UPDD demonstrates auto-monitoring + pre-merge gates catch errors test suites miss
- Halt-and-recovery is normal operation, not failure
- Each catch produces backlog item (PR), preventing recurrence

**Empirical evidence**:
- Cron #41 catch (Step 5.5 cache incident): silent functional failure
- Coder pre-merge halt (Option C v1): structural inconsistency
- Both errors caught **before** production contamination

**Why publishable separately**:
- Verification protocol design is rarely emphasized in MM-PBSA literature
- Two real-world catches in one development cycle is strong evidence
- Reproducible pattern: other groups can adopt similar architecture

### 7.3 Methodology Contribution: Phased Versioning Framework

The process-oriented vs outcome-oriented distinction (§4.2.1) is unusual in field practice.

**Novel angle**:
- v0.7 = process-oriented (methodology setup quality)
- v0.8/0.9 = outcome-oriented (methodology output quality)
- Most projects conflate the axes, producing over-claims or under-claims
- Phased separation makes incremental contributions explicit

**Why publishable angle**:
- Field rarely separates "infrastructure correct" from "predictions accurate"
- UPDD demonstrates that they are independent contributions worth distinguishing
- Honest versioning enables progress measurement without over-claiming

### 7.4 Honest Disclosure as Strength (Pre-patch X.A re-evaluation)

The Phase α process **could have** claimed pre-patch X.A UNCONDITIONAL as a quantitative match:
- "Cp4 ΔΔG = +3.92, CI95 includes both Magotti -1.4 and -3.0"
- This was technically true at the time

**Phase α revealed** this was methodologically conditioned:
- Charge bias contributing ~3.8 kcal/mol to ΔΔG
- Post-patch reveals genuine PBSA + ff14SB regime
- Lucky-or-not unverifiable, but conditioned status documented

**Novel angle for paper 1**:
- Most papers claim positive results, not document re-evaluation
- UPDD's honesty about pre-patch interpretation is a transparency contribution
- This is harder for reviewers to challenge than a clean "we got the answer"

**This is not over-claimed humility**: it's methodologically rigorous self-correction.
